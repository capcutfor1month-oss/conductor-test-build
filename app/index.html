Connection Setup
1
Start Conductor Bridge
Not Running
The Bridge is a tiny local server (port 4611) that lets Conductor talk to Ableton, NotebookLM, and the audio analyzer. Keep it running whenever you use Conductor.
double-click tools/start_bridge.sh in Finder
Or double-click tools/start_bridge.sh from Finder.
2
Open Ableton Live with MCP
Not Connected
Make sure Ableton Live is open and the MCP remote script is loaded. Ableton must be set to:

Prefs → MIDI → Control Surface 1 = Ableton_Live_MCP
If Ableton MCP isn't responding, try:
pkill -f ableton-live-mcp
Then reconnect in Claude Code. Ableton will auto-retry on next message.
3
NotebookLM CLI
Not Found
Conductor uses notebooklm-py to query your personal music knowledge base. Install it once:
pipx install notebooklm-py
If installed at a custom path, set it here:
Click Auto-detect to find your installation
4
Plugin Bridge VST3
Load in Ableton
Plugin Bridge lets Conductor control Pro-Q 4, Serum 2, and any third-party plugin. Load it in Ableton:
1. In Ableton, create an Audio or MIDI track
2. Drag PluginBridge.vst3 onto the track
3. In the plugin UI, select the channel name and pick a plugin
4. MCP connects automatically — no restart needed
VST3 location: /Library/Audio/Plug-Ins/VST3/PluginBridge.vst3
